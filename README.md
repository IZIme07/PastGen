# PastGen

**PastGen** — приложение для создания, исследования и сохранения семейной истории.
Вы не заполняете формы — вы загружаете разрозненные материалы (фото, документы,
PDF, заметки), а ИИ-генеалог превращает хаос в **проверяемое, живое семейное дерево**.
Каждый факт объясняет, откуда он взялся; неуверенность видима, но не пугает;
конфликты становятся исследовательскими задачами.

Дерево, таймлайн и карта — три равноправных взгляда на одну историю.

## Возможности

- **Дерево / Граф** — панорамирование, зум к курсору, перетаскивание узлов,
  переключение «классическое дерево ↔ граф связей», ветви цветом, кольца уверенности.
- **Досье человека** — утверждения с источниками, уверенностью, альтернативными
  версиями и конфликтами; Markdown-биография; исследовательские задачи.
- **Импорт хаоса** — загрузка фото/PDF/текста; ИИ извлекает имена, даты, места,
  связи и создаёт **предложения**. Импорт никогда не меняет дерево сам:
  каждое предложение вы принимаете, откладываете или отклоняете.
- **Таймлайн** — полосы жизни по годам, поколения, направляющие эпох, линия «сегодня».
- **Карта** — реальная карта (OpenStreetMap) в тёплой архивной тонировке:
  места семьи и маршруты миграций.
- **ИИ-генеалог** — исследовательский помощник с задачами: «проверь ветку»,
  «найди слабые места», «письмо в архив», «вопросы для интервью» и др.
- **Проверка противоречий** — рождение после смерти родителя, слишком юный родитель,
  пересекающиеся браки, дубли — автоматически становятся задачами, а не ошибками.

## Быстрый старт (продакшн, Docker)

```bash
cp .env.example .env        # заполните ANTHROPIC_API_KEY (необязательно) и BASIC_AUTH
docker compose up --build -d
# приложение: http://localhost:8080
```

Данные (SQLite-база и загруженные файлы) живут в `./data` — этого каталога
достаточно для резервной копии: `tar -czf backup.tgz data/`.

Демо-данные (семья Соколовых) внутри контейнера:

```bash
docker compose exec pastgen node -e "1" # контейнер работает?
docker compose run --rm pastgen node dist/seed.js   # либо локально: npm run seed
```

## Разработка

Требуется Node.js ≥ 22.

```bash
npm install
npm run seed     # демо-данные (семья Соколовых) в ./server/data
npm run dev      # сервер на :8081 + Vite на :5173 (прокси /api)
npm test         # vitest: сервер (домены, API) + клиент (компоненты)
npm run build    # production-сборка клиента и сервера
```

## Конфигурация (.env)

| Переменная | Назначение |
|---|---|
| `ANTHROPIC_API_KEY` | Ключ Anthropic API. Без него приложение полностью работает, но ИИ-функции (извлечение фактов, генеалог) отключены и показывают спокойную подсказку. |
| `ANTHROPIC_MODEL` | Переопределить модель (по умолчанию `claude-opus-4-8`). |
| `PORT` | Порт сервера (по умолчанию 8080). |
| `DATA_DIR` | Каталог данных: SQLite + загрузки (по умолчанию `./data`, в Docker — `/data`). |
| `BASIC_AUTH` | `пользователь:пароль` — включает HTTP Basic Auth. Обязательно для публичного доступа. |

## Архитектура

```
client/   React 18 + Vite + TS — дизайн-система PastGen (токены и компоненты
          портированы из design/), рабочая область: дерево, таймлайн, карта, импорт
server/   Express + better-sqlite3 + zod — REST API, движок предложений
          (транзакционное применение), движок противоречий, ИИ-модуль (Anthropic)
shared/   общие TypeScript-типы (только типы, без рантайма)
design/   дизайн-система из Claude Design — источник визуальной истины
docs/     функциональная спецификация, спека и план реализации
```

Ключевые принципы (из спецификации):

- **Каждый факт — утверждение** (claim) с источниками, уверенностью 0–100
  и альтернативными версиями. Альтернативы не теряются.
- **ИИ всегда предлагает, пользователь подтверждает.** Импорт пишет только
  в слой предложений; принятие применяется одной транзакцией с записью в историю.
- **Противоречия — это данные**, а не ошибки: они помечают конфликты и создают задачи.

## Ограничения v1 (осознанные)

- **Аудио** сохраняется как источник, но не транскрибируется автоматически
  (Claude API не делает speech-to-text); распознанный текст можно внести вручную.
- Распознавание лиц, видеоистории, совместная работа родственников и экспорт
  PDF-книги — вне MVP (спецификация §17.2).
- Один пользователь / одна семья на развёртывание; для публичного доступа
  включите `BASIC_AUTH` и поставьте TLS-прокси (Caddy/nginx) сверху.
- Уязвимости из `npm audit` относятся только к dev-инструментам (vite/esbuild)
  и не попадают в производственный образ.
